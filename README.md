# openclaw-openclaw

This repository is organized as an internal AI skills library following the Anthropic Skills Repository model.
It is designed to host reusable AI tools, connectors, and helper scripts in a central, browsable format.

## Quick Start

1. Clone and stay updated
   ```bash
   git clone https://github.com/dsm-developer/openclaw-openclaw.git
   cd openclaw-openclaw
   git pull origin main
   ```
2. Link the library to your local AI environment
   - Open the repository in VS Code and allow your AI assistant / Copilot workspace access.
   - For local agents like Cursor or other AI tools, point them at this folder and ask them to inspect `/skills`.
3. Contribute a new tool
   - Copy `/skills/_template` into a new folder under `/skills`.
   - Update `SKILL.md`, `logic.py`, and `README.md` for your new skill.
   - Commit your changes and open a PR against `main`.

## Repository Structure

- `/skills`: Contains all AI tools and connectors.
- `/skills/_template`: A reusable tool blueprint for new contributions.
- `/skills/smartsuite-connector`: Proof-of-concept SmartSuite connector.
- `.github/workflows/update-skill-catalog.yml`: GitHub Action that refreshes the Skill Catalog on every push to `main`.
- `.gitignore`: Blocks environment files, credentials, and common secret file patterns.

## Contributing

1. Use `/skills/_template` as your starting point.
2. Make sure secrets are only read from environment variables.
3. Add a concise human-readable `README.md` and maintain a clear `SKILL.md`.
4. Submit a PR to `main`; branch protection should require review and PR validation.
